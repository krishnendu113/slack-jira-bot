🤖 Slack JIRA Assistant Bot Overview

This Slack-integrated bot simplifies JIRA issue management by combining semantic understanding, smart field validation, and multi-tool orchestration — all running as a lightweight Vercel serverless function. It provides a seamless user experience with minimal back-and-forth, using Slack message history as short-term memory to maintain context.

Figure: High-level architecture of the Slack JIRA Assistant Bot on Vercel

🚀 Key Features

  • 🔍 Smart Duplicate Ticket Detection
    Uses semantic search via OpenAI embeddings and JIRA keyword-based search. Summarizes and shares results.
  • 👤 Post-Creation Optional Assignment
    Allows assignment after creation if requested. Uses email ID to identify possible assignees.
  • 📝 One-Step Field Gathering & Validation
    Collects and validates all necessary fields in one step, offering suggestions based on allowed values.
  • 🧾 Scoped, Helpful, and Polite
    Answers only JIRA-related queries and gracefully declines unrelated requests.
  • 🔐 Privacy-First by Design
    Never reveals internal tool details or user data; access is strictly scoped to JIRA ticket actions.
  • 🛠️ Agent-Driven Multi-Tool Execution
    Agent can retrieve tickets, validate fields, create issues, and assign tickets — all with minimal user interaction.
  • 🧠 Slack-Based Conversational Memory
    Maintains recent message history from Slack to provide context-aware responses.
  • 📚 Retrieval-Augmented Generation (RAG)
    Connects to Supabase for vector search and summarizes matches using GPT-4.1-mini.
  • ⚙️ Deployed as a Vercel Serverless Function
    Lightweight, fast, and scalable with zero maintenance overhead.

🔁 Data Flow

  • User Sends Message in Slack → Bot is triggered by new user message.
  • Message History Retrieved → Fetches recent Slack messages for memory.
  • Intent Understanding & Field Extraction → Determines user intent and required tools.
  • Parallel Search Operations → Runs semantic and text-based searches to find similar tickets.
  • Field Collection & Validation → Validates and confirms all inputs with allowed JIRA values.
  • JIRA Ticket Creation → Creates ticket after gathering complete, validated data.
  • Optional Ticket Assignment → Assigns issue post-creation if user requested it.
  • Response Posted Back to Slack → Shares links and status updates with user in Slack.
